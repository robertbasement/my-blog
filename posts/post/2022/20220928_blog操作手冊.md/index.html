Hugo指令

新增文章

hugo new post/<md file path>

這樣就可以在content/post/下看到新創的markdown檔

文章調整

基本上我所有的文章都是從jupyter轉換過來
在用hugo new post create好md檔之後
得把jupyter轉換過的md檔複製到
hugo格式中筆記的位置 需要run 這個指令under root (content folder的上一層)

jupyter nbconvert --to markdown --output-dir=<output dir> <ipynb file>

有一個問題是 轉換後圖會被存在另一個folder中
假設你的jupyter file叫 test.ipynb 在轉換的output dir中還會有一個folder 叫 test_files.ipynb 此時可以把你的圖片都丟到 static/images中 並用以下command line
將圖片的url做修改

sed -i '' 's/your_notebook_files\//\/images\//g' your_notebook.md

另一個問題是 jupyter轉md下，dataframe的表現會跑掉
在每一個notebook中可能有多個dataframe
故得在notebook最上方執行以下的python code 將每一個dataframe轉換成html 就可以在轉成md後正常顯示了

from IPython.display import HTML
import pandas as pd

def display_df_as_html(df):
    """Automatically display DataFrame as an HTML table."""
    display(HTML(df.to_html()))

# Apply function to all DataFrame outputs
pd.DataFrame._repr_html_ = display_df_as_html

開幾本地server

hugo server

先在本地測試預覽用的

打包成靜態網頁檔(每次更新過blog內容後)

hugo -D

執行完後，打包的靜態網頁會輸出在public/資料夾下

GitHub Page相關

  1. 在root的.gitignore 要加上 public/
  2. git init public folder
cd public
git init
git remote add origin https://github.com/username/my-hugo-site.git
  1. push to github
git add .
git commit -m "Deploy Hugo site"
git branch -M main
git push origin main

Note 若是網站有更新過後 要回到root 並重跑

hugo -D

之後重複step 3.
(不用在git init, 即使hugo -D 會更新public folder的內容 但不是全部刪除 所以第一次的git init 還是能track)
